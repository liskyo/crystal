# 專案架構及程式碼分析

## 1. 專案概述 (Project Overview)

本專案名為 **CrystalAI (ROPU PON 晶飾)**，這是一個結合**傳統面相學 (Physiognomy)** 與 **現代人工智慧 (AI)** 技術的 Web 應用程式。

### 核心功能
*   **面相能量掃描**：使用者上傳照片或使用鏡頭捕捉面部影像。
*   **AI 智能分析**：利用 Google Gemini 模型分析面部特徵點，結合使用者輸入的姓名、性別及期望能量（Intent）。
*   **水晶頻率匹配**：根據 AI 分析的能量場 (Aura) 與面相特徵，從系統內建的水晶資料庫中推薦最合適的水晶飾品。
*   **個人化報告**：提供詳細的靈氣分析報告、面相特徵解讀及能量啟示。

---

## 2. 技術架構 (Technical Architecture)

本專案採用現代化的前端與伺服器無伺服器 (Serverless) 架構：

*   **前端框架 (Frontend Framework)**: [React](https://react.dev/) + [Vite](https://vitejs.dev/)
    *   使用 React Hooks (`useState`, `useEffect`) 進行狀態管理。
    *   使用 TypeScript 確保型別安全。
*   **樣式與 UI (Styling & UI)**: [Tailwind CSS](https://tailwindcss.com/)
    *   採用 Utility-first CSS 進行快速且響應式的介面開發。
    *   包含豐富的動畫效果 (`animate-in`, `animate-pulse` 等) 提升使用者體驗。
*   **AI 服務 (AI Service)**: [Google Gemini API](https://ai.google.dev/)
    *   使用 `@google/genai` SDK 與 Gemini 模型互動。
    *   主要使用模型：`gemini-3-flash-preview` 與 `gemini-1.5-flash`（具備自動降級/重試機制）。
*   **後端/API (Backend/API)**: Vercel Serverless Functions
    *   `/api/analyze`：處理核心分析邏輯，保護 API Key 不暴露於前端。
*   **資料儲存 (Data Storage)**:
    *   **靜態資料**：水晶產品列表儲存於 `constants.tsx`。
    *   **本地狀態**：部分暫存狀態 (如訪客數) 使用 `localStorage`。

## 4. 核心程式碼詳細分析 (Code Deep Dive)

### 4.1. 主應用邏輯 (`App.tsx`)

這是整個應用程式的指揮中心，管理著用戶的使用流程狀態 (`AppState`)。

*   **狀態機 (State Machine)**:
    *   `IDLE`: 初始畫面，顯示專案介紹與開始按鈕。
    *   `INPUT`: 輸入個人資料 (姓名、性別、期望能量)。
    *   `SCANNING`: 選擇上傳照片或開啟鏡頭。
    *   `ANALYZING`: 顯示分析中的載入動畫。
    *   `RESULT`: 展示 AI 分析結果與推薦水晶。
*   **關鍵邏輯**:
    *   `handleCapture`: 接收圖片並呼叫 `analyzeFaceAndCrystal` 服務。
    *   `useEffect`: 模擬並管理「訪客計數器」，將數據保存在 `localStorage`。
    *   UI 渲染：根據 `state` 條件渲染對應的畫面區塊。

### 4.2. AI 分析介面 (`api/analyze.ts`)

這是系統最核心的「大腦」，負責將圖片轉化為有意義的玄學分析。

*   **安全與配置**:
    *   支援多組 API Key (`process.env.GEMINI_API_KEY`)，並實作隨機負載平衡 (Load Balancing) 避免單一 Key 達到速率限制。
*   **Prompt Engineering (提示詞工程)**:
    *   精心設計的 System Prompt 將 AI 設定為「面相學與能量水晶专家」。
    *   **性別邏輯**：特別針對男性、女性及其他性別設定了不同的水晶推薦偏好（例如：男性避免過於粉嫩的款式，除非心輪需要療癒）。
    *   **結構化輸出**：要求 AI 返回嚴格定義的 JSON 格式，包含：
        *   `faceDetected`: 是否偵測到人臉 (防呆機制)。
        *   `faceTraits`: 面相特徵分析列表。
        *   `auraColor`: 能量場顏色。
        *   `energyReading`: 詩意的能量判讀。
        *   `recommendationReason`: 推薦理由。
*   **容錯機制 (Fallback Strategy)**:
    *   優先嘗試最新的 `gemini-3-flash-preview` 模型。
    *   若遇上 Rate Limit (429) 或其他錯誤，會自動降級至 `gemini-1.5-flash` 模型重試。

### 4.3. 資料結構 (`types.ts`)

定義了前後端溝通的標準格式，確保資料一致性。

*   `UserInfo`: 使用者基本資料 (Name, Gender, Intent)。
*   `CrystalProduct`: 水晶產品規格 (ID, Name, Description, Benefit, Tags, Price, Image, ExternalLink)。
*   `AnalysisResult`: AI 返回的分析結果結構。

### 4.4. 水晶資料庫 (`constants.tsx`)

*   包含 `CRYSTAL_PRODUCTS` 陣列，儲存了所有待售的水晶手鍊資訊。
*   每項產品都包含詳細的描述、功效標籤 (`tags`) 和購買/外部連結 (`externalLink`)。
*   包含 `INTENTS` 列表，定義了使用者可以選擇的 12 種能量祈願（如：事業、愛情、健康等）。

## 5. 亮點功能與設計細節 (Highlights)

1.  **沈浸式體驗 (Immersive Experience)**:
    *   大量使用 Backdrop Blur (毛玻璃效果)、漸層與呼吸燈動畫 (`animate-pulse`)，營造神秘且高級的視覺感受。
2.  **人性化的 AI 互動**:
    *   不僅僅是隨機推薦，而是真的將使用者的「面相特徵」與「水晶功效」做邏輯綁定，讓使用者感受到「被理解」。
3.  **穩健的系統設計**:
    *   後端考慮了 API Key 輪替與模型降級策略，大大提升了服務的穩定性。
    *   前端 `try-catch` 完整處理錯誤，並給予使用者友善的提示（如：照片不清晰時的提示）。

## 6. 潛在優化建議 (Future Optimization)

*   **圖片壓縮**: 目前在前端將 Base64 傳輸給後端，若圖片過大可能影響傳輸速度。可考慮在前端先進行 Canvas 壓縮。
*   **快取機制**: 對於相同的分析結果可以做短暫快取，節省 API Token 消耗。
*   **資料庫整合**: 目前產品資料是寫死的 (Hardcoded)，未來可串接 CMS (如 Contentful 或 Strapi) 或資料庫 (PostgreSQL/MongoDB)，以便動態管理商品庫存與價格。
